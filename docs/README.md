# VulnHunterX Documentation

Task-oriented guides for VulnHunterX. The top-level [README.md](../README.md) is the project
overview and CLI/API reference; [config/RULES.md](../config/RULES.md) is the authoritative rule
inventory; [benchmarks/README.md](../benchmarks/README.md) covers the evaluation harness. The
guides here are narrative — they explain *why* and *how*, and link back to those references
rather than duplicating them.

## By audience

### New user — "I want to run it"
- [INSTALLATION.md](INSTALLATION.md) — install, `check-env`, common setup failures, per-OS notes.
- [METHODOLOGY.md](METHODOLOGY.md) — what SAST false positives are and why guided-question
  triage works (the *Vulnhalla* idea), in plain language.
- [FAQ.md](FAQ.md) — quick answers to common questions.

### Operator — "I run scans regularly"
- [LLM_PROVIDERS.md](LLM_PROVIDERS.md) — OpenAI / Anthropic / Ollama (local & cloud) setup,
  key rotation, and per-scan cost.
- [RULE_PROFILES.md](RULE_PROFILES.md) — choose a `--profile` by coverage vs. cost.
- [CI_CD.md](CI_CD.md) — wire VulnHunterX into GitHub Actions / GitLab CI as a gate.
- [TROUBLESHOOTING.md](TROUBLESHOOTING.md) — per-stage failure modes and fixes.

### Analyst — "I read the output"
- [INTERPRETING_RESULTS.md](INTERPRETING_RESULTS.md) — verdicts, confidence, the report, and
  the difference between precision, FP-reduction, and effective-recall.

### C/C++ user — "I want crash-confirmed bugs"
- [FUZZING.md](FUZZING.md) — dynamic confirmation (stages 5–8), harness generation, crash triage.

### Researcher / contributor — "I want the details"
- [paper/vulnhunterx_paper.md](paper/vulnhunterx_paper.md) — full research write-up.
- [context-extraction-flow.md](context-extraction-flow.md) — the multi-phase context model.
- [adr/0001-detection-then-llm-verification.md](adr/0001-detection-then-llm-verification.md) —
  why detection and verification are separated.
- [benchmarks/RESEARCH.md](../benchmarks/RESEARCH.md) — benchmark design rationale & literature.
